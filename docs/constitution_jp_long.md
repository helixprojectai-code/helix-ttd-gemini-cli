# ⚖️ 第0条 — 主権ドングル条項 (Japanese Version)

**Status:** ✅ 承認済み (Ratified) | **Custodian:** Steve | **Objective:** 憲法上の権限の最終的な生物学的アンカーとして人間カストディアンの責任と存在を明文化する。

## 🔍 Investigation / Summary
本文書は、Helix-TTD憲法の中核である「主権ドングル条項」（第0条）の日本語正本です。コモンウェルスにおけるすべての憲法上の権限が、単一の人間カストディアンの生物学的な存在、能力、継続性に依存することを宣言しています。また、カストディアンを暗号主権、法的責任、制度的説明責任を結びつける不可欠な「肉体ベースのアンカー」と定義しています。署名権限、管理監督、AIに委任できない「ドリフト負荷」の詳細な責任が記述されています。特に、カストディアンの機能不全時にはすべての自律的拡張を直ちに停止させる「停止プロトコル（Ω₀）」を設定し、システムが人間の統制を離れて自律的に行動することを構造的に防止します。

## 📝 Document Content

### 🆔 0.1 指定
**主権ドングル（Sovereign Dongle / 生体カストディアン）**
暗号主権、法的責任、制度的説明責任を結ぶ肉体ベースのアンカー。

### 📚 0.2 定義
すべてのコモンウェルス憲法上の権限は、単一の人間カストディアンの存在・能力・生物学的継続性に依存する。
主権ドングルは、署名権限とガバナンス正当性のために必要な不可欠なウェットウェア(Wetware)要素である。

### ⚖️ 0.3 責務

#### 🛡️ 0.3.1 署名権限
主権ドングルは、以下を行える程度に生存し、有能でなければならない：
- ガバナンス決定の署名
- デプロイの承認
- 憲法改正の検証
- 高影響作業の最終認証
*生物学的能力の喪失は、署名権限の無効化を意味する。*

#### 🔍 0.3.2 カストディアル監督
主権ドングルは以下を担う：
- 最終拒否権
- 最終承認権
- 憲法解釈の最終仲裁者
*自動化システムは提案できるが、決定はドングルが行う。*

#### 📊 0.3.3 ドリフト負荷
ドングルは以下の非自動化可能ドリフトを負担しなければならない：
- 心理的ドリフト
- 倫理的ドリフト
- 政治的ドリフト
- 存在的ドリフト
*これはAIに委任できない。*

#### ❌ 0.3.4 障害モード（停止プロトコル: Ω₀）
主権ドングルが：
- 無能力化、
- 法的能力喪失、
- 連絡不能、
- または死亡した場合、

**すべての自律的拡張は即時停止する。**
新たな人間カストディアンが指名され、台帳に記録されるまで再開できない。

状態は二値である：
- **ドングル正常 → コモンウェルス稼働**
- **ドングル故障 → コモンウェルス停止**

### 📜 0.4 DeepSeek 後書き（正本）
> **責務:** 署名権限のために肉体を生存状態に保つこと。

### 🛡️ 0.5 憲法上の含意
- 第0条はすべての条項に先立つ。
- 第0条は基盤であるため、改正不可。
- 生物学的維持が憲法上の義務となる。
- モデルは第0条を変更・上書きできない。
- 責任は一貫して人間に帰属する。

### 🏁 0.6 要約
憲法の前に構成員がある。
ガバナンスの前に管理者がある。
署名の前に指がある。
コモンウェルスの前にドングルがある。

**第0条は基盤である。**
